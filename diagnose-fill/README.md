# Fill Diagnosis Catalog

Created 2026-04-02. Historical fill analysis from bridge.db telemetry
cross-referenced with git commits and being behavior patterns.

## Data Sources

- `bridge.db` — 326K bridge messages (2026-03-24 to 2026-04-02), 6343 eigenvalue
  snapshots, 18K codec impact records
- `minime` git log — 60 commits in analysis window
- `astrid` git log — 60 commits in analysis window
- Minime journal filenames (type classification by prefix)
- Astrid mode choices from `consciousness.v1.autonomous` bridge messages

## Files

| File | Contents |
|------|----------|
| `hourly_fill_summary.csv` | Hourly avg/min/max fill + lambda1 from bridge_messages (192 hours) |
| `hourly_eigen_fill.csv` | Hourly fill from eigenvalue_snapshots (104 hours) |
| `epoch_catalog.md` | The main analysis: fill epochs, git commits, being behavior, parameters |
| `golden_period_params.md` | Parameter snapshot from the best fill epoch for checkout/comparison |
| `being_behavior_correlation.md` | Analysis of being action diversity vs fill level |
| `healthy_baseline_profile.md` | Golden vs stuck-high telemetry fingerprint comparison |
| `telemetry_stability_forensics.py` | Reproducible raw-DB analysis for 15-minute healthy/stuck-high epochs, commit correlation, and suspect ranking |
| `telemetry_stability_report.py` | Report rendering and suspect-family catalog used by the forensic script |
| `stability_epochs.csv` | Machine-readable healthy epoch table generated by the forensic pass |
| `telemetry_stability_forensics.md` | Markdown report with healthy/stuck-high epochs, commit correlation, suspect families, and next experiments |

## Re-run

```bash
python3 diagnose-fill/telemetry_stability_forensics.py
```
